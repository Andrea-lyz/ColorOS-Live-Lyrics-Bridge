# ColorOS Live Lyrics Bridge v2.7.0

这一版在 v2.6.0 的外部 Provider 接入基础上，继续收紧锁屏歌词渲染、屏幕超时保活和酷狗/网易云 Provider 的切歌链路，重点是更稳、更安静、更可控。

## 新增

- 新增歌词界面设置页，可单独控制普通逐行歌词的伪逐字高亮、翻译行逐字高亮、滚动放大、非实时行模糊。
- 新增锁屏屏幕保活设置，可关闭智能保活，也可设置亮屏保持秒数；留空或 `0` 表示歌词界面显示期间不额外限制保活时长。
- LyricProvider 外部协议新增 Provider 声明字段，Provider 可以主动声明播放器包名、播放状态、曲目 generation、当前曲目可信度和翻译按钮能力。
- 酷狗 Provider 新增 MediaSession `lyricInfo` 主动发布链路，Bridge 可在 SystemUI 尚未收到外部广播时直接从当前媒体会话读取完整歌词。

## 优化

- 优化屏幕超时保活语义：保活重新支持从可见官方歌词行兜底识别锁屏歌词界面，减少 SystemUI 日志格式变化或日志延迟导致的保活失效；自定义超时只控制保活窗口，wake lock 仍保持短租约续租，降低异常长持有风险。
- 降低默认运行日志噪声：普通信息日志默认静默，酷狗 Provider 诊断日志仅在显式 debug tag 下输出。
- 优化 SystemUI 性能开销：缓存歌词 RecyclerView 类型判断与 TextView 所属 RecyclerView 关系，减少高频绘制与布局路径上的父节点遍历和字符串匹配。
- 优化官方歌词行缩放与模糊过渡，降低 AOD/亮屏切换、外部 Provider 接管和播放跳转时的旧行闪现与缩放抖动。
- 优化外部 Provider 切歌逻辑，使用声明式播放器身份、generation、trackKey 和当前曲目可信度过滤迟到歌词，减少跨曲目串词。
- 优化酷狗 Provider：歌词文件读取增加写入完成等待、本地歌词文件兜底探测、候选歌词延迟绑定和缓存命中校验，切歌时更少出现“暂无歌词”或上一首歌词残留。
- 优化网易云 Provider：根据用户当前歌词模式决定是否发送逐字 `rawLyric`，避免普通歌词模式下合成逐字时间轴带来的显示节奏偏差。
