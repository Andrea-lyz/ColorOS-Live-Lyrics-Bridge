# ColorOS Live Lyrics Bridge v3.0.0

这是一次面向锁屏歌词完整链路的大版本更新。相较 v2.7.2，本版重构了 Bridge 与各 LyricProvider 的切歌、歌词到达和播放状态交接，修复多类“暂无歌词”、旧歌词残留、逐字进度错位和 AOD 切换抖动问题，为偶发 SystemUI 崩溃增加定向保护，并进一步降低 SystemUI 绘制热路径与调试日志的开销。

## 重大更新

- 统一 QQ 音乐、网易云音乐、Apple Music、Poweramp、Spotify、汽水音乐和酷狗音乐 Provider 的 Bridge 事件模型：使用曲目 generation 区分 `trackChanged` 与 `lyricReady`，对需要外部状态的 Provider 继续区分 `playbackState`，丢弃异步返回的旧曲歌词，并对重复 payload 和高频播放状态广播去重。
- 重做外部歌词切歌交接：在新曲歌词尚未到达、播放器 metadata 短暂抖动或 SystemUI RecyclerView 正在换行绑定时，保持曲目身份和渲染状态一致，减少“暂无歌词”、上一首歌词闪回、边界外首句和淡化未完成等问题。
- 稳定亮屏到 AOD 的歌词绘制：沿用 SystemUI 官方首帧 prime 和布局时序，降低 AOD 低帧率模式下的连续刷新与复杂进度效果，修复靠近绘制区域下边界的歌词异常抖动。
- 新增严格限定在 `LyricsRecyclerView` 的 RecyclerView Adapter 通知保护，仅拦截布局计算期间的对应 `IllegalStateException`，降低切歌和状态变化时 SystemUI 偶发崩溃风险。

## 播放器修复

- ConePlayer：兼容“媒体通知（车载蓝牙）歌词”复用媒体元数据的情况，避免临时歌词行被误判为切歌后令锁屏显示“暂无歌词”，同时保留原有车载歌词功能。
- Spotify：改善首次歌词到达等待、快速切歌、播放进度接续和无歌词曲目处理；当前仍仅支持原文歌词，暂不支持翻译。
- QQ 音乐：修复切歌后旧歌词偶发重复、首句越界，以及快速切歌后歌词淡化或渲染状态未完整恢复的问题。
- 网易云音乐：补齐曲目 generation、播放状态和旧异步结果过滤，降低快速切歌时的错绑风险。
- Apple Music：修复暂停时歌词区域闪现、部分歌曲长时间“暂无歌词”、显示文本与逐字数据不一致及逐字进度错位；增加匹配 PlaybackItem 的请求重试，并兼容交叉渐入渐出期间的曲目交接。
- Poweramp：补齐显式 `trackChanged`、`lyricReady` 事件并复用一次生成的歌词 payload；播放状态保持由原生 MediaSession 单路提供；规范化方括号行内逐字时间戳，并在官方索引短暂落后时让放大行回跟实际进度，避免拖动或恢复播放后歌词越界、滚动行与高亮行错位。
- 汽水音乐：优化歌词对象、缓存和播放状态的接管顺序，修复部分有歌词歌曲仍显示“暂无歌词”，并让切歌交接更自然。
- 酷狗音乐/概念版：过滤缺少有效曲目身份的早期播放状态，保留稳定 generation 与歌词 payload，降低启动和切歌竞态。

## 性能与日志

- 减少 SystemUI `TextView.onDraw` 热路径中的重复父级查找、可见区域计算、时间读取和临时对象分配；相同歌词行的最近绘制缓存改为原位刷新。
- 普通逐行歌词在不需要逐帧进度效果时自动降低刷新频率；AOD 模式继续使用更保守的刷新节奏。
- `Official lyric row scale` 仅记录目标状态变化，不再在动画的每一帧重复输出；详细诊断仍可通过对应 debug tag 显式开启。
- QQ、网易云、Apple Music、Poweramp、Spotify、汽水和酷狗 Provider 的 Bridge 成功/跳过日志统一受 debug tag 控制，发布版默认只保留必要警告和错误；Poweramp 的内部反射探针仅在显式开启 debug tag 后安装。
