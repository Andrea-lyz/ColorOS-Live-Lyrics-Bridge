# ColorOS Live Lyrics Bridge v3.4.0

v3.4.0 将 Bridge 与配套 LyricProvider 统一到新的歌词传递链路，重点是让完整歌词、逐字进度、翻译与切歌状态在 ColorOS 锁屏上更稳定地协同工作。

## Bridge

- Bridge 主模块新增专属应用图标，替换此前的系统默认图标。
- 统一外部歌词 v4 传递协议，并按已登记的播放器与来源接收数据，减少切歌、后台恢复和播放器进程重载后的歌词丢失。
- 优化外部歌词的轨道切换、播放进度与翻译状态衔接；锁屏翻译按钮在播放器提供翻译时保持可用。
- 修复酷狗 Lite / 酷狗概念版部分逐字歌词行的渲染错位与空白问题。
- 优化逐字进度扫光和已播放文字的过渡，使高亮变化更连贯自然。
- 改进歌词开头清理：规则与逐曲“第一句正式歌词”不再被旧容量限制截断，跨歌词轨道时能更可靠地保留用户选择的首句。

## LyricProvider

- 网易云音乐 Provider 兼容历史 9.0.40 的 `:play` 播放进程及荣耀版包名；对内置 LSPatch / DexKit 的特殊 APK 使用兼容初始化，避免冲突。
- 修复 QQ 音乐 QRC 原文与翻译时间戳存在偏差时的翻译行遗漏，改为按时序一对一对齐，并保留“无翻译”行的语义。
- 新增 **LX Music** Provider，支持 ToSide 与 Walnut 两个 LX Music 变体；可传递完整时间轴和可用的翻译歌词。
- QQ 音乐、网易云音乐、Apple Music、Poweramp、Spotify、汽水音乐、酷狗音乐/概念版与 LX Music 共 8 个 Provider 已统一适配 v4 链路。
- 保留 LyricProvider / 词幕原有功能；本次只调整面向 Bridge 的传递与诊断路径。

## 升级说明

- 请将 Bridge 与已安装的 Provider 一起更新，避免混用旧版 Provider。
- 首次安装或升级后请重启设备，使 SystemUI 与播放器进程重新加载模块。
