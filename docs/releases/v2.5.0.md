# ColorOS Live Lyrics Bridge v2.5.0

这一版重点修复复杂歌词解析和官方锁屏渲染稳定性，尤其是逐字歌词、翻译、罗马音、重复歌词和格式异常混在一起时的错位问题。

## 核心更新

- 新增歌词 lane 分类器，同时间多行会稳定区分主歌词、翻译、罗马音、原文变体和 credit 行。
- 修复 `He did it / 他背叛了我` 这类异常逐字时间导致的夹心显示问题，避免渲染成 `He did / 他背叛了我 / it`。
- 修复 Bejeweled、coney island 等增强 LRC 中主歌词被翻译行覆盖，只显示翻译的问题。
- 修复部分歌词缺失翻译、重复短句、多句同时间戳时的 lane 错位和漏解析。
- 清理“以下歌词翻译由 xxx 提供”等翻译提供方说明，避免被当作歌词内容。
- 对异常大间隔或非递增的逐字时间做整行降级，减少坏格式歌词触发的渲染异常。
- 修复 All Too Well 后段重复歌词和翻译近邻导致的锁屏歌词抖动。
- 修复 closure、willow、'tis the damn season 等开头 title/credit 行抢占首句焦点的问题，第一句歌词在正式开始前保持清晰。

## 锁屏显示与设置

- 新增歌词界面设置页，可切换默认样式、滚动放大和非实时行模糊。
- 优化官方歌词列表 alias 复用和首句 display anchor，降低官方列表缺 title/credit 或行号偏移时的错位风险。
- 优化行高、缩放、模糊和 AOD/锁屏切换时的刷新稳定性。

## 性能与日志

- 默认关闭逐帧布局诊断和解析 trace，减少常规使用时的 logcat 噪声。
- 保留 `LockscreenLyrics` 和 `LockscreenLyricsParse` DEBUG log tag，用于后续精准排查。
- 优化长歌词热路径查询，减少每帧重复扫描歌词行。

## 测试

- 新增歌词解析回归矩阵，覆盖 no body, no crime、Bejeweled、coney island、provider credit 清理、异常逐字时间降级等场景。
- 发布前执行 `:app:testDebugUnitTest` 和 `:app:assembleDebug`。
