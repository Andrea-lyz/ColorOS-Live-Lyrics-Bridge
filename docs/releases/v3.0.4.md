# ColorOS Live Lyrics Bridge v3.0.4

v3.0.4 完善了关闭“普通逐行伪逐字”后的长歌词显示：普通逐行歌词换行超过两行时，不再固定截断在前两行，而是在保持两行视窗和既有 RecyclerView 几何不变的前提下，随整句播放进度平滑展示后续内容。

## Bridge 改进

- 普通逐行歌词关闭伪逐字后，三行正文使用固定两行裁剪视窗，从第 1、2 行平滑移动至第 2、3 行。
- 被动滚动完全由当前歌词的行时间轴驱动，不额外模拟字级时间，也不绘制伪逐字高亮。
- 滚动采用 `21.5%～78.5%` 的有效进度区间和五次平滑插值，起步、停止更柔和，中段移动更自然。
- 暂停时滚动位置冻结，恢复后继续；拖动播放进度时直接落到对应位置，避免独立动画时钟造成漂移。
- 仅在符合条件的当前长歌词需要移动时保持显示帧率刷新；普通短句和静止状态继续走较低刷新频率。
- 重复或回收中的非当前歌词 holder 不会覆盖当前行的刷新判断。

## 稳定性边界

- 不改变 `LyricsRecyclerView` 的滚动位置、item 定位、prime 或 `setCurrentLyric` 时机。
- 不修改普通 SystemUI `TextView`；裁剪和自绘仍只作用于歌词 RecyclerView 内部。
- AOD 继续沿用已验证的低帧率与首次 attach 时序，不启用本次连续被动滚动。

## LyricProvider

- LyricProvider 代码与内部版本保持不变，已安装的 Provider 无需更新；Release 仍附带各播放器 Provider APK 和合集，便于首次安装或补装。
